<Callout type="warn" title="Planned Content">
**Status:** New page

This guide will cover all the ways to set up signers for your client: **`payer(signer)`** to provide an explicit `TransactionSigner`, **`payerFromFile(path)`** to load a keypair from a JSON file (Node.js only, e.g. Solana CLI keypair), **wallet standard integration** for using a browser wallet as the signer, **`generatedPayer()`** to generate a random keypair for testing, **`generatedPayerWithSol(amount)`** to generate a signer and airdrop funds to it for testing, **`payerOrGeneratedPayer(payer?)`** to use an explicit signer if provided or auto-generate one otherwise, and how the signer flows through transaction planning and signing.

**Draws from:** Current `getting-started/signers.mdx` (signer generation and airdrop sections), `@solana/kit-plugin-payer` README.

**Links to:** [Advanced Guides — Signers](/docs/advanced-guides/signers) for the signer interface deep dive.
